Mongoose CRUD Tutorial

Creation and Usage of Schemas

The first step to building a Mongoose CRUD application is creating a schema!

First, create a folder called 'models' that will store your schema.

Mongoose schemas are similar to MongoDB schemas with slightly shortnened syntax:

const postSchema = new mongoose.Schema({
   
    title: String,
    tag1: String,
    tag2: String,
    tag3: String,
    subject: String,
    content: String,
    username: String,
    userID: {
      type: mongoose.Schema.Types.ObjectId,
      ref: 'User',
    },
    image: String,
    createdAt: {
        type: Date,
        default: new Date()
    }
});

As shown above, when you create your schema you must declare it as a variable ( 'postSchema' ) and then call the Mongoose .Schema method.

At the top of your page, remember to declare 'mongoose' and 'Schema' as constant variables:

const mongoose = require('mongoose');

const Schema = mongoose.Schema;

Finally, after doing so and modeling your schema, declare a constant variable to name your collection. By default, Mongoose will take this variable, pluralize it and name your collection after:

const Post = mongoose.model('Post', postSchema);
 
module.exports = Post;

Set Up of Routes and Database Connection

The next step to setting up our API is the index.js page.

Create this file inside the root folder of your API.

At the top of the page, you must require all your necessary modules you will be using: 

const bodyParser = require('body-parser');
const express = require('express');
const mongoose = require('mongoose');

const app = new express();

Directly below, require any files you need such as schemas and controllers ( more on that later )

const Post = require('./database/models/Post');
const storePostController = require('./controllers/storePost')

Next, create your Mongoose connection string and add some simple error checking while you're at it:

mongoose.connect('mongodb://localhost:27017/admin', { useNewUrlParser: true })

    .then(() => 'You are now connected to Mongo!')
    .catch(err => console.error('Something went wrong', err))

Now you may use Express syntax to create any routes you may need.

app.get("/", homePageController);
app.post("/posts/store", auth, storePostController);

Finally, tell Express to listen on a port, then run 'node' or 'nodemon' to start up your API.

app.listen(4000, () => {
    console.log('App listening on port 4000')
});
